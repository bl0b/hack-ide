hack-ide by bl0b

A simple wrapper to tmux that enables custom and PERSISTENT work environments in a (nut)shell.

I love my terminals, and I used to SSH+screen a lot. My problem with screen is : - the multiplexing is not great, - I lose my buffer and history when the remote machine reboots, or if my screen or command gets killed. So I created this wrapper that recreates my work sessions, preserving shell histories, vim session, etc... A session is easily described in a short text file that defines the context (session) name, the tasks to be run, and their layout in one or many windows, hack-ide supports many levels of templating to shorten even further the description files, and all RC files related to a session are easily identified by hack-ide, and, as easily, versionable.

Dependencies

tmux, python, cmake (for automated packaging only)

Install

$ cd ./package then $ ./build.sh && dpkg -i hack*deb or $ cmake -DCMAKE_INSTALL_PREFIX=/usr && make install
